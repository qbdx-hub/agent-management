# Agent 管理系统 — 答辩问题准备

## 一、整体架构

| 问题 | 回答要点 |
|------|---------|
| 为什么选 Spring Boot + Vue3？ | 前后端分离、Spring Boot 生态成熟、Vue3 组合式 API 开发效率高、Element Plus 组件库丰富 |
| 前后端怎么通信的？ | RESTful API + JWT Token + Axios 拦截器统一加 Authorization / X-Workspace-Id 头 |
| 多租户怎么隔离的？ | 每个请求带 X-Workspace-Id 头，后端 SecurityUtils.currentWorkspaceId() 取值，所有查询都加 workspaceId 条件 |
| 用的什么数据库？为什么？ | MySQL 8.0，关系型数据库适合结构化数据，MyBatis-Plus 简化 CRUD |
| 后端分层怎么设计的？ | Controller → Service → ServiceImpl → Mapper，VO 返回前端、Form 接收参数、Entity 映射数据库 |

---

## 二、Auth 认证模块

| 问题 | 回答要点 |
|------|---------|
| 登录流程是什么？ | 前端 POST /auth/login → 后端校验密码(BCrypt) → 生成 JWT → 前端存 localStorage → 后续请求带 Bearer token |
| JWT 怎么校验的？ | JwtAuthenticationFilter 拦截请求 → 解析 token → 从 Redis 查 session → 设置 SecurityContext |
| 密码怎么存储的？ | BCrypt 单向哈希，数据库存哈希值，不存明文 |
| Token 过期怎么处理？ | 前端拦截器 401 时清 localStorage 跳登录页；后端 JwtAuthenticationFilter 校验过期时间 |
| 注册流程？ | POST /auth/register → 校验用户名/邮箱唯一 → BCrypt 加密密码 → 插入 user 表 → 创建默认工作空间 |

---

## 三、Agent 管理模块

| 问题 | 回答要点 |
|------|---------|
| Agent 表有哪些关键字段？ | 基本信息 + 模型配置(provider/model/temperature/maxTokens) + 提示词 + 记忆策略 + 执行配置 + AI 连接配置(baseUrl/apiKey/model) |
| Agent 状态机怎么设计的？ | draft → testing → published → paused → archived，前端下拉选择，后端校验合法值集合 |
| 怎么保存 Agent 配置的？ | PUT /agents/{id}，MyBatis-Plus updateById 只更新非 null 字段（部分更新语义），一个接口支持所有配置更新 |
| 知识库怎么绑定到 Agent？ | Agent.knowledgeBaseIds JSON 数组存知识库 ID，记忆策略页面多选绑定 |
| Agent 的 CRUD 怎么做的？ | 标准 RESTful：GET 列表(分页)、GET 详情、POST 创建、PUT 更新、PATCH 状态、DELETE 删除 |

## 四、AI 对话模块

| 问题 | 回答要点 |
|------|---------|
| 怎么接入真实 AI 的？ | OkHttp 调 OpenAI 兼容 API（/v1/chat/completions），用户在 Agent 配置页填 Base URL / API Key / 模型名，不绑定固定供应商 |
| 流式回复怎么实现的？ | 后端 SseEmitter 发送 SSE 事件(thinking/content/done)，前端 XMLHttpRequest + setInterval 轮询读取 responseText |
| 对话上下文怎么构建的？ | system prompt + 知识检索结果 + 最近 N 轮历史消息(workingWindow 可配) + 当前用户消息 |
| 为什么不用 WebSocket？ | SSE 更简单、浏览器原生支持、单向推送足够用（服务端→客户端）、HTTP 天然兼容 |
| 消息怎么存储的？ | message 表存每条消息(role/content/token统计)，session 表存会话元数据(状态/token总量/费用) |
| 没配置 AI 会怎样？ | 返回模拟回复，提示用户去 Agent 配置页填写 AI API 信息 |

## 五、知识库模块

| 问题 | 回答要点 |
|------|---------|
| 文档处理流程是什么？ | 上传 → 存盘 → 点"处理" → 提取文本 → 按 chunk_size 滑窗分块(带 overlap) → 调 embedding API 生成向量 → 存 document_chunk 表 |
| 向量检索怎么实现的？ | 用户问题 → 生成 query embedding → 从 MySQL 加载所有 chunk 向量 → Java 层计算余弦相似度 → 返回 topK |
| 为什么用 MySQL 存向量？ | 项目规模小、chunk 数量有限、不引入外部依赖、应用层计算性能可接受 |
| 没有 embedding 配置怎么办？ | 关键词回退：把用户问题拆成 2-4 字滑窗关键词，用 SQL LIKE 模糊匹配 OR 条件 |
| 支持什么文件格式？ | txt / md / json / csv / yaml 等纯文本（PDF/DOCX 需要额外解析库，暂不支持） |
| 分块参数怎么配的？ | chunk_size 默认 500 字符、overlap 默认 50 字符，可在知识库 config JSON 中自定义 |

## 六、工具管理模块

| 问题 | 回答要点 |
|------|---------|
| 工具怎么调用的？ | 工具表存 endpoint_url / method / headers / parameters，Agent 绑定工具后可被 AI function calling 使用 |
| 工具分类有哪些？ | search / compute / operate / perceive / notify / custom，按功能域划分 |
| MCP 工具是什么？ | Model Context Protocol，标准化的 AI 工具协议，支持 stdio / sse 两种传输方式 |
| 工具认证怎么做？ | auth_type 字段(none/api_key/bearer/oauth)，auth_config JSON 存认证配置 |

## 七、Workflow 编排模块

| 问题 | 回答要点 |
|------|---------|
| 画布怎么实现的？ | Vue Flow 库实现节点拖拽连线，节点类型：start / agent / tool / condition / end |
| 节点配置怎么保存的？ | 画布 JSON（节点位置/连线/配置）序列化存 workflow.canvas_json 字段 |
| Agent/Tool 节点怎么选资源？ | 点击添加时弹对话框，调 API 获取已有 Agent/Tool 列表，选择后关联 ID |
| condition 节点怎么处理的？ | 保留字转义，支持条件分支路由 |

## 八、监控面板模块

| 问题 | 回答要点 |
|------|---------|
| 监控数据从哪来的？ | 概览从 session/message 表聚合（活跃 Agent 数、今日调用、成功率、延迟） |
| 告警怎么配置的？ | alert_rule 表存规则(metric/threshold/duration)，触发后生成 alert_record |
| Token 趋势怎么统计的？ | 按天/小时 GROUP BY 聚合 session 表的 total_tokens 字段 |

## 九、成本管理模块

| 问题 | 回答要点 |
|------|---------|
| 费用怎么统计的？ | 每次 AI 调用生成 cost_record（token 数 × 单价），按 model / agent / user 维度聚合 |
| 预算怎么控制的？ | budget 表存限额/周期/阈值，overview 计算使用百分比，超 80% 警告超 90% 熔断 |
| 预估月底费用怎么算的？ | 本月已花费 ÷ 已过天数 × 本月总天数 |
| 消费明细怎么查的？ | GET /cost/records 分页查询 cost_record 表，按 recordedAt 降序 |

## 十、审计日志模块

| 问题 | 回答要点 |
|------|---------|
| 怎么记录操作日志的？ | AOP @AuditLog 注解 + AuditLogAspect 切面，声明式标注在 Controller 方法上 |
| 记录哪些信息？ | 用户 ID / 用户名 / 操作类型 / 资源类型 / 资源 ID / IP / UA / 时间 / 结果 |
| SpEL 表达式是干什么的？ | 从方法参数中动态提取 resourceId（如 #id、#docId），解决多参数场景下定位资源 |
| 审计失败会影响业务吗？ | 不会，Aspect 内 try-catch 吞异常并 log.error，不阻断业务方法执行 |

## 十一、安全设计

| 问题 | 回答要点 |
|------|---------|
| 怎么做权限控制的？ | Spring Security + RBAC 四角色(ADMIN/MANAGER/DEVELOPER/VIEWER)，URL 级别权限配置 |
| API Key 怎么存储的？ | 哈希存储，前端只显示脱敏后的前 8 位 + *** |
| SQL 注入怎么防护的？ | MyBatis-Plus 参数化查询，不拼接 SQL |
| XSS 怎么防护的？ | 前端 v-html 只渲染受控 Markdown，后端 Jackson 默认转义特殊字符 |

---

## 十二、数据库设计

| 问题 | 回答要点 |
|------|---------|
| 哪些字段用了 JSON？ | agent.tags / prompt_variables / knowledge_base_ids / output_schema、session.variables、message.attachments、document_chunk.embedding、budget.notify_channels |
| 为什么用 JSON 字段？ | 灵活存储数组/对象，避免额外关联表，MyBatis-Plus JacksonTypeHandler 自动序列化 |
| 分页怎么实现的？ | MyBatis-Plus Page 对象 + 分页插件，前端传 page/pageSize，后端返回 list/total/page/pageSize |
| 表之间怎么关联的？ | 外键逻辑关联（不建物理外键），如 session.agent_id → agent.id、message.session_id → session.id |

## 十三、部署相关

| 问题 | 回答要点 |
|------|---------|
| 前端怎么打包的？ | Vite build → dist 静态文件，开发环境 vite dev proxy 到后端 8080 |
| 后端怎么启动的？ | mvn spring-boot:run 或打 fat jar，内嵌 Tomcat，端口 8080，context-path /api/v1 |
