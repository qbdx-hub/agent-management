# 第一章  概述
## 1.1 设计背景
近年来，以大语言模型（LLM）为核心的人工智能技术取得了突破性进展，基于大模型的 AI Agent（智能体）正在从实验室走向产业落地。AI Agent 能够自主理解用户意图、调用外部工具、执行多步推理，在代码生成、数据分析、文档处理、客服问答等场景展现出强大的应用潜力。然而，当一个企业或团队同时运营数十甚至上百个 Agent 时，传统的命令行配置和脚本管理方式暴露出明显的短板：配置分散难以统一管理、缺乏可视化的调试手段、工具和知识库无法复用、成本和安全难以管控。
在此背景下，业界急需一个统一的平台，将 Agent 的创建、配置、调试、监控、编排、治理等环节集成在一起，让开发者和团队能够像管理传统软件资产一样管理 AI Agent。本项目"Agent 数字资产管理系统"正是基于这一需求而设计的，旨在为团队提供一个可视化的 Agent 全生命周期管理平台，让 AI 能力可控、可管、可复用。
## 1.2 设计目的及意义
本系统的设计目的是构建一个覆盖 AI Agent 全生命周期的管理平台，具体包括以下目标：第一，提供可视化的 Agent 创建与配置界面，降低 Agent 开发门槛；第二，支持工具市场和知识库复用，避免重复建设；第三，提供多 Agent 编排能力，实现复杂业务场景的自动化；第四，提供监控、成本、安全等治理能力，保障 Agent 的稳定运行。
本系统的意义体现在三个层面：对个人开发者，告别命令行和配置文件，通过 UI 快速搭建和调试 Agent，提升开发效率；对技术团队，统一管理所有 Agent，复用知识库和工具，实现团队协作开发；对企业管理者，掌握 Agent 成本、安全、合规情况，让 AI 资产透明可控。本系统的设计与实现，对于推动 AI Agent 技术在企业中的规模化应用具有一定的实践参考价值。
## 1.3 相关现状
目前市面上已有多款 AI Agent 管理与编排平台，具有一定代表性的产品包括：Dify，开源的 LLM 应用开发平台，支持 Prompt 编排、RAG、Agent 编排，社区活跃但企业级治理能力较弱；Coze（扣子），字节跳动推出的 Agent 搭建平台，面向个人开发者，提供可视化编排但不开源；FastGPT，开源的知识库问答系统，侧重 RAG 场景，Agent 编排能力有限；LangFlow，基于 LangChain 的可视化编排工具，灵活但缺乏完整的管理后台。
与上述产品相比，本系统的差异化定位在于：第一，面向团队而非个人，内置工作空间、成员管理、权限控制等团队协作能力；第二，覆盖 Agent 全生命周期，从创建到监控到退役形成闭环；第三，内置成本管理和安全审计模块，满足企业级治理需求；第四，采用前后端分离架构，技术栈主流（Vue 3 + Spring Boot），便于二次开发和私有化部署。
# 第二章  相关技术介绍
## 2.1 前端技术栈
本项目前端采用 Vue 3 框架为核心，搭配一系列现代化前端工具链，构建高性能、可维护的单页应用（SPA）。主要技术选型如下：
Vue 3.5：采用 Composition API 和 <script setup> 语法，相较于 Options API，逻辑组织更灵活，代码复用性更强。Vue 3 的响应式系统基于 Proxy 实现，性能优于 Vue 2 的 Object.defineProperty 方案。
TypeScript 6.0：全量使用 TypeScript，为所有变量、函数、组件 props 声明类型，在编译期捕获潜在错误，提升代码健壮性和开发体验。配合 vue-tsc 进行类型检查，确保类型安全。
Vite 8.0：新一代前端构建工具，基于浏览器原生 ES 模块实现极速冷启动和热更新（HMR），开发体验远优于传统 webpack。生产构建使用 Rollup 打包，支持 Tree Shaking 和代码分割。
Element Plus 2.14：基于 Vue 3 的企业级 UI 组件库，提供表格、表单、弹窗、菜单等 60+ 组件，开箱即用，大幅减少 UI 开发工作量。
Pinia 2.3：Vue 官方推荐的状态管理库，相较于 Vuex 更轻量，支持 TypeScript，API 设计更符合 Composition API 风格。本项目使用 5 个 Store 管理用户、工作空间、Agent、会话、全局 UI 状态。
Vue Flow 1.48：基于 Vue 3 的流程图编辑器库，用于 Agent 编排模块的可视化画布，支持节点拖拽、连线、缩放、小地图等功能。
此外还使用了 Axios（HTTP 客户端）、Vue Router 4（路由）、ECharts（图表）、Monaco Editor（代码编辑器）、@microsoft/fetch-event-source（SSE 流式通信）等库。
## 2.2 后端技术栈
本项目后端采用 Java + Spring Boot 为主的技术栈，提供 RESTful API 服务。主要技术选型如下：
Spring Boot 3：基于 Spring Framework 的快速开发框架，内置 Tomcat 服务器，自动配置机制大幅减少 XML 配置。Spring Boot 3 要求 JDK 17+，支持 GraalVM 原生镜像。
MyBatis-Plus 3.5：MyBatis 的增强工具，在不影响原生功能的前提下提供 CRUD 自动生成、分页插件、代码生成器等便利功能，减少样板代码。本项目使用 BaseMapper 自动生成单表 CRUD，复杂查询手写 XML Mapper。
MySQL 8.0：开源关系型数据库，支持 JSON 字段类型、窗口函数、CTE 等特性。本项目数据库字符集统一为 utf8mb4，共设计 30 张数据表，使用 JSON 字段存储 Agent 的复杂配置信息。
Redis 7：内存数据库，用于缓存热点数据、分布式锁、会话管理等场景。本项目使用 Redis 缓存 Agent 配置和用户会话，减轻数据库压力。
JWT（JSON Web Token）：用于用户认证，登录成功后服务端签发 Token，前端在后续请求的 Header 中携带 Token，服务端拦截器校验有效性。相较于 Session 方案，JWT 无状态、易扩展。
SSE（Server-Sent Events）：用于 Agent 流式输出，服务端通过长连接逐步推送生成内容到前端，实现打字机效果。相较于 WebSocket，SSE 更轻量，适合单向推送场景。
# 第三章  需求分析与概要设计
## 3.1 需求分析
经过对 AI Agent 管理场景的调研和分析，本系统的需求分为功能需求和非功能需求两部分。
功能需求方面，系统需要支持 AI Agent 的全生命周期管理，具体包括十大功能模块：Agent 工作台（全局概览与快速入口）、Agent 配置中心（创建、配置、版本管理）、工具与插件管理（工具注册、测试、绑定）、Agent 编排引擎（可视化工作流设计）、记忆与知识库（文档管理与 RAG 检索）、会话与任务控制台（实时对话与执行追踪）、监控与可观测性（健康监控与告警）、成本与资源管理（费用统计与预算）、安全与治理（权限审计与审批）、团队与工作空间（成员管理与协作）。系统对外提供 74 个 RESTful API 接口。
非功能需求方面：性能上，API 平均响应时间不超过 500ms，前端首屏加载不超过 3 秒；安全上，所有接口需 JWT 鉴权，敏感操作需审计日志，支持 RBAC 权限模型；可扩展性上，采用模块化设计，新增模块不影响现有功能；可用性上，支持 Mock 模式，前端可独立于后端开发和演示。
## 3.2 概要设计
### 3.2.1 总体设计
本系统采用前后端分离架构。前端 Vue 项目独立部署，通过 Axios 调用后端 RESTful API；后端 Spring Boot 项目提供 API 服务，访问 MySQL 数据库和 Redis 缓存，并通过 SSE 实现 Agent 流式输出。前端开发服务器默认端口 5173，后端服务默认端口 8080，API 统一前缀为 /api/v1。
后端采用经典分层架构：Controller（接口层）负责接收 HTTP 请求和返回响应；Service（业务层）处理核心业务逻辑；Mapper（数据访问层）通过 MyBatis-Plus 操作数据库；Entity（实体层）定义数据结构。各层通过依赖注入解耦，职责清晰。
前端采用模块化架构：api 层封装所有 HTTP 请求；stores 层使用 Pinia 管理全局状态；views 层按业务模块组织页面组件；components 层存放可复用组件；router 层配置路由和权限守卫。每个业务模块（如 agent、tool、orchestration）拥有独立的 api、store、views，模块间低耦合。
### 3.2.2 功能模块设计
系统共设计 10 个功能模块，各模块职责如下：
（1）Agent 工作台：用户进入系统的首页，展示 Agent 卡片列表、统计数据、最近任务、团队动态，提供全局搜索和快速创建入口。
（2）Agent 配置中心：Agent 的核心管理模块，支持 Agent 的增删改查、模型配置、System Prompt 编辑（含版本管理）、工具绑定、记忆策略配置等。
（3）工具与插件管理：管理 Agent 可调用的外部工具，支持 API 工具注册、MCP 服务器接入、内置工具管理，提供在线测试面板。
（4）Agent 编排引擎：通过可视化画布设计多 Agent 协作流程，支持 Agent 节点、条件节点、审批节点等，工作流以 JSON 格式存储，支持运行记录回放。
（5）记忆与知识库：管理 Agent 的记忆策略，支持文档上传、向量化、RAG 检索配置。
（6）会话与任务控制台：与 Agent 实时对话，支持 SSE 流式输出、执行步骤可视化、错误追踪。
（7）监控与可观测性：监控 Agent 健康状态、Token 消耗趋势，配置告警规则，提供链路追踪。
（8）成本与资源管理：统计各 Agent 和模型的费用，配置预算上限和告警阈值。
（9）安全与治理：基于 RBAC 的角色权限管理，审计日志记录敏感操作，高风险操作审批流程，API Key 管理。
（10）团队与工作空间：工作空间隔离，成员管理，工作空间级配置。
## 3.3 数据库设计
本系统数据库采用 MySQL 8.0，字符集 utf8mb4，共设计 30 张数据表。整体结构以工作空间（Workspace）为核心，每个工作空间下管理 Agent、工具、工作流、知识库等资源，通过 workspace_id 实现数据隔离。主要数据表按模块分组如下：
认证权限模块（3 张表）：user 用户表、role 角色表、user_role 用户角色关联表，实现 RBAC 权限模型。工作空间模块（2 张表）：workspace 工作空间表、workspace_member 成员表。Agent 管理模块（3 张表）：agent 表（含 model_config、memory_config 等 JSON 字段）、agent_prompt_version 提示词版本表、agent_tool_binding 工具绑定表。工具市场模块（2 张表）：tool 工具表、tool_call_record 调用记录表。会话控制模块（4 张表）：session、message、execution_step、error_log。监控告警模块（2 张表）：alert_rule、alert_record。成本管理模块（2 张表）：cost_record、budget。安全审计模块（4 张表）：audit_log、approval_rule、approval、api_key。编排引擎模块（4 张表）：workflow、workflow_node、workflow_edge、workflow_run。知识库与通用模块（4 张表）：knowledge_base、document、activity_log、model_pricing。
数据库设计中大量使用 JSON 字段（如 agent 表的 model_config、workflow 表的 definition），将复杂的配置结构以 JSON 形式存储，避免了过多的关联表，在查询灵活性和存储效率之间取得了平衡。所有表均包含 created_at 和 updated_at 时间戳字段，由 MyBatis-Plus 的 MetaObjectHandler 自动填充。
# 第四章  详细设计与实现
## 4.1 开发环境介绍
本项目的开发环境配置如下：
操作系统：Windows 11。后端开发工具：IntelliJ IDEA 2024.1，支持 Spring Boot 项目开发、Maven 依赖管理、数据库连接、断点调试。前端开发工具：Visual Studio Code，搭配 Volar（Vue 语言支持）、TypeScript Vue Plugin 等扩展。数据库管理：Navicat Premium 16，用于 MySQL 数据库的可视化管理、SQL 执行、数据导入导出。版本管理：Git 2.40+，代码托管于 Gitee，采用 main/dev/feature 分支策略。接口测试：Postman，维护接口集合和环境变量。运行环境：JDK 17、Node.js 22、Maven 3.8、MySQL 8.0、Redis 7。
## 4.2 数据库链接（完成人：童倍）
本项目后端通过 Spring Boot 的自动配置机制连接 MySQL 数据库，采用 Druid 数据库连接池管理连接。数据库连接信息配置在 application.yml 文件中，主要包括数据源 URL、用户名、密码以及连接池参数。Druid 连接池提供了监控统计、SQL 防火墙、慢 SQL 日志等功能，便于性能调优和问题排查。
数据源配置的核心代码如下（application.yml）：
【YAML 配置】
spring:
datasource:
type: com.alibaba.druid.pool.DruidDataSource
driver-class-name: com.mysql.cj.jdbc.Driver
url: jdbc:mysql://10.7.17.154:3306/agent_management
?useUnicode=true&characterEncoding=utf-8
&serverTimezone=Asia/Shanghai
username: dev
password: dev123456
druid:
initial-size: 5
min-idle: 5
max-active: 20
max-wait: 60000
validation-query: SELECT 1
test-while-idle: true
上述配置中，url 指向团队共用的 MySQL 数据库（部署在 10.7.17.154 主机的 3306 端口），字符编码设为 utf-8 以支持中文，时区设为 Asia/Shanghai。连接池初始大小 5，最小空闲 5，最大活跃连接 20，超时等待 60 秒。validation-query 配置 SELECT 1 用于连接有效性检测，test-while-idle 开启空闲连接检测，避免使用已断开的连接。
在 MyBatis-Plus 配置方面，开启了驼峰命名自动映射（map-underscore-to-camel-case），使数据库的 snake_case 字段自动映射为 Java 的 camelCase 属性。同时配置了逻辑删除字段（deleted），删除操作自动转为更新 deleted=1，避免物理删除数据。
## 4.3 功能模块一：工作台（完成人：童倍）
工作台是用户进入系统的首页，需要展示全局信息并提供快速入口。本模块实现了 Agent 卡片列表、统计概览、最近任务列表、快速创建 Agent 等功能。
实现方法：工作台页面 DashboardView.vue 采用三栏布局，左侧展示"我的 Agent"卡片列表，中间展示"最近任务"，右侧展示"团队动态"。Agent 卡片列表通过调用 /api/v1/agents 接口分页获取当前工作空间下的 Agent，以 el-card 组件渲染，每张卡片显示名称、描述、状态标签、模型信息和最近调用时间。统计数据通过聚合接口获取 Agent 总数、活跃数、今日任务数、Token 消耗，使用 ECharts 渲染趋势图表。快速创建功能点击按钮后弹出 QuickCreate 对话框，引导用户三步创建 Agent。
工作台前端 API 调用的关键代码如下（src/api/agent.ts）：
【TypeScript】
import http from './index'
import type { ApiResponse, PaginatedData, PaginationQuery } from '@/types/common'
import type { AgentSummary } from '@/types/agent'
// 分页获取 Agent 列表（工作台卡片数据来源）
export async function getAgentList(
params: PaginationQuery
): Promise<ApiResponse<PaginatedData<AgentSummary>>> {
const res = await http.get<ApiResponse<PaginatedData<AgentSummary>>>(
'/agents', { params }
)
return res.data
}
// 获取 Agent 详情
export async function getAgentDetail(
id: number
): Promise<ApiResponse<AgentDetail>> {
const res = await http.get<ApiResponse<AgentDetail>>(`/agents/${id}`)
return res.data
}
上述代码中，getAgentList 函数接收分页参数（page、pageSize、keyword、status），通过 axios 发送 GET 请求到后端 /agents 接口，返回分页数据。TypeScript 泛型确保了返回类型的正确性。在 Mock 模式下，函数会从本地 mock 数据返回，无需后端即可开发调试。
实现结果：工作台页面能正确展示 Agent 卡片列表，支持搜索筛选；统计图表实时反映 Agent 使用情况；快速创建弹窗能引导用户完成 Agent 创建。页面加载时间约 1.2 秒，交互流畅。
## 4.4 功能模块二：Agent 管理（完成人：童倍）
Agent 管理是系统最核心的模块，包含 Agent 的全生命周期管理。本模块实现了 Agent 列表、创建向导、详情配置（含提示词编辑、工具绑定、记忆配置）等功能。
实现方法：AgentList.vue 使用 el-table 展示分页列表，支持按名称搜索和按状态筛选。AgentCreate.vue 实现三步向导式创建：第一步填写基本信息（名称、描述、头像、标签），第二步配置模型（选择供应商、模型版本、温度、最大 Token），第三步确认提交，使用 el-steps 组件控制步骤切换。AgentDetail.vue 采用 Tab 布局，包含基础配置、提示词、工具绑定、记忆配置四个 Tab。PromptEditor.vue 集成 Monaco Editor 编辑 System Prompt，支持变量插入和版本回滚，每次保存自动创建版本记录存入 agent_prompt_version 表。
Agent 创建接口的关键代码如下（后端 AgentController.java）：
【Java】
@RestController
@RequestMapping("/api/v1/agents")
public class AgentController {
@Autowired
private AgentService agentService;
// 创建 Agent
@PostMapping
public Result<Agent> create(@RequestBody @Valid CreateAgentDTO dto) {
Long workspaceId = SecurityUtils.getCurrentWorkspaceId();
Agent agent = agentService.create(dto, workspaceId);
return Result.success(agent);
}
// 分页查询 Agent 列表
@GetMapping
public Result<IPage<AgentSummary>> list(AgentQueryDTO query) {
IPage<AgentSummary> page = agentService.pageList(query);
return Result.success(page);
}
// 更新 Agent 状态
@PatchMapping("/{id}/status")
public Result<Void> updateStatus(
@PathVariable Long id,
@RequestBody @Valid UpdateStatusDTO dto) {
agentService.updateStatus(id, dto.getStatus());
return Result.success();
}
}
上述代码中，AgentController 使用 Spring MVC 的注解式路由，@PostMapping 处理创建请求，@GetMapping 处理列表查询。@Valid 注解触发参数校验，CreateAgentDTO 中定义了 @NotBlank、@Size 等校验规则。SecurityUtils.getCurrentWorkspaceId() 从 JWT Token 中解析当前工作空间 ID，实现数据隔离。Service 层封装业务逻辑，通过 MyBatis-Plus 的 BaseMapper 操作数据库。
实现结果：Agent 列表分页加载流畅，搜索筛选响应迅速；创建向导三步流程清晰，表单校验完善；提示词编辑器支持语法高亮和版本管理，能一键回滚历史版本。整个模块共 7 个页面组件，是系统中页面数量最多的模块。
## 4.5 功能模块三：工具市场（完成人：童倍）
工具市场用于管理 Agent 可调用的外部工具，是 Agent 能力扩展的关键入口。本模块实现了工具列表、工具注册、工具详情与测试等功能。
实现方法：ToolMarket.vue 以卡片网格形式展示所有工具，每个工具卡片显示名称、描述、类型标签（API/MCP/内置）、调用次数和状态，支持按类型筛选。ToolRegister.vue 提供表单式注册界面，填写工具名称、描述、请求方法、URL、请求头、参数 Schema（JSON Schema 格式），使用 ParamSchemaEditor 组件辅助编辑 JSON Schema。ToolDetail.vue 展示工具详情并提供 ToolTestPanel 测试面板，用户输入参数后实时调用工具查看返回结果，调用记录写入 tool_call_record 表用于统计。
工具注册的前端表单提交关键代码如下（src/views/tool/ToolRegister.vue）：
【Vue 3】
<script setup lang="ts">
import { reactive, ref } from 'vue'
import { ElMessage } from 'element-plus'
import { createTool } from '@/api/tool'
import type { CreateToolDTO } from '@/types/tool'
const formRef = ref()
const submitting = ref(false)
const form = reactive<CreateToolDTO>({
name: '',
description: '',
type: 'api',
config: {
method: 'GET',
url: '',
headers: {},
paramSchema: {},
responseExample: {}
}
})
async function handleSubmit() {
await formRef.value.validate()
submitting.value = true
try {
await createTool(form)
ElMessage.success('工具注册成功')
router.push('/tool/market')
} catch (e) {
ElMessage.error('注册失败：' + (e as Error).message)
} finally {
submitting.value = false
}
}
</script>
上述代码使用 Vue 3 的 Composition API，reactive 创建响应式表单对象，ref 创建提交状态。handleSubmit 函数先调用表单校验（validate），通过后调用 createTool API 提交，使用 try-catch 处理异常，finally 恢复按钮状态。ElMessage 提供用户反馈。整个流程遵循"校验-提交-反馈"的标准模式。
实现结果：工具列表展示清晰，类型筛选和搜索功能正常；工具注册表单校验完善，JSON Schema 编辑器支持语法高亮；测试面板能实时调用工具并展示返回结果，方便开发者验证工具配置是否正确。
## 4.6 功能模块四：Agent 编排（完成人：童倍）
Agent 编排是系统最有技术含量的模块，允许用户通过可视化画布设计多 Agent 协作流程。本模块实现了工作流列表、可视化编辑器、运行记录等功能。
实现方法：WorkflowEditor.vue 是本模块的核心，使用 Vue Flow（@vue-flow/core）实现画布拖拽编辑。页面分为三栏：左侧节点面板提供开始、Agent、工具、条件判断、人工审批、结束等可拖拽节点类型；中间画布支持节点拖拽放置、连线、缩放、平移；右侧 NodeEditor 面板编辑选中节点的属性。用户从左侧拖拽节点到画布，节点间通过 Handle（连接点）拉线形成工作流。工作流结构序列化为 JSON 存入 workflow 表的 definition 字段，包含 nodes 数组和 edges 数组。运行时后端解析 definition，按拓扑顺序执行各节点，通过 SSE 推送执行进度到前端。
Vue Flow 画布初始化与节点连线的关键代码如下（src/views/orchestration/WorkflowEditor.vue）：
【Vue 3 + Vue Flow】
<script setup lang="ts">
import { ref } from 'vue'
import { VueFlow, useVueFlow, type Connection } from '@vue-flow/core'
import { Background } from '@vue-flow/background'
import { Controls } from '@vue-flow/controls'
import { MiniMap } from '@vue-flow/minimap'
const nodes = ref<any[]>([])
const edges = ref<any[]>([])
const { onConnect, addEdges, onNodeClick, onPaneClick } = useVueFlow()
// 节点连线：从源节点右侧 Handle 拉到目标节点左侧 Handle
onConnect((connection: Connection) => {
addEdges({
...connection,
id: 'e_' + Date.now(),
type: 'smoothstep',
data: { condition: null }
})
})
// 点击节点选中，点击空白取消
onNodeClick(({ node }) => { selectedNode.value = node })
onPaneClick(() => { selectedNode.value = null })
// 拖拽添加节点到画布
function addNode(nodeType: string, ref?: { id: number; name: string }) {
const newNode = {
id: 'n_' + Date.now(),
type: 'custom',
position: { x: Math.random() * 300, y: Math.random() * 200 },
data: {
label: NODE_LABELS[nodeType],
nodeType,
agentId: ref?.id,
agentName: ref?.name
}
}
nodes.value = [...nodes.value, newNode]
}
</script>
<template>
<VueFlow v-model:nodes="nodes" v-model:edges="edges"
:default-viewport="{ zoom: 1 }">
<Background pattern-color="#aaa" :gap="16" />
<Controls />
<MiniMap />
</VueFlow>
</template>
上述代码中，useVueFlow() 返回的 onConnect 回调在用户拉线连接两个节点时触发，addEdges 将新边加入 edges 数组，type 设为 smoothstep 使连线呈阶梯状。addNode 函数在用户拖拽节点到画布时调用，生成包含唯一 id、位置坐标和业务数据的节点对象。VueFlow 组件通过 v-model 双向绑定 nodes 和 edges，Background、Controls、MiniMap 分别提供背景网格、缩放控制和小地图。
实现结果：可视化编辑器支持节点拖拽、连线、缩放、平移、小地图等完整交互；节点属性编辑面板能实时同步选中节点的配置；工作流保存后能正确序列化为 JSON 存储；运行记录页面能展示各节点执行状态和耗时。该模块是系统中交互最复杂、技术含量最高的模块。

# 第六章  结论
本项目设计并实现了一个 Agent 数字资产管理系统，覆盖了 AI Agent 从创建、配置、编排到监控、治理的全生命周期。系统采用 Vue 3 + Spring Boot 前后端分离架构，包含 10 个功能模块、74 个 API 接口、30 张数据表，实现了 Agent 可视化管理、工具市场、多 Agent 编排、监控告警、成本管理、安全审计等核心能力。
本人负责的工作台、Agent 管理、工具市场、Agent 编排四个模块是系统的核心功能链路，涵盖了从数据展示到业务配置到可视化编排的完整能力。其中 Agent 编排模块使用 Vue Flow 实现的可视化工作流编辑器，是本项目的技术亮点，支持用户通过拖拽连线的方式设计多 Agent 协作流程，降低了编排门槛。
通过本项目的开发，验证了前后端分离架构在中等复杂度管理系统中的适用性，Vue 3 的 Composition API 配合 TypeScript 显著提升了前端代码的可维护性，Spring Boot 的自动配置和 MyBatis-Plus 的 CRUD 生成大幅提高了后端开发效率。同时，JSON 字段在 MySQL 中的灵活运用，为存储复杂配置结构提供了简洁的方案。
系统仍有改进空间：一是缺少自动化测试体系，后续应引入 JUnit 和 Vitest 建立测试驱动开发流程；二是编排引擎的执行能力有限，目前仅支持顺序和条件分支，后续可扩展循环、并行等执行模式；三是缺少大模型对接的实际运行验证，目前以 Mock 模式为主，后续需接入真实大模型 API 进行端到端验证。
